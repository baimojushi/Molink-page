Project overview

给神经系统
一个安静的接口。

Meditation Tent 是一套交互装置系统。早期版本通过腰带压力传感和 HRV 手环探索生理输入与反馈映射，后期引入毫米波雷达做非接触呼吸检测，最终通过灯光与实时视觉构建空间级反馈。HRV 手环始终保留——它的角色是效能验证：评估装置是否帮助使用者降低焦虑、提升副交感神经活跃度，形成"体验层 + 验证层"的双层架构。
01

项目的核心判断

多数冥想类装置停留在视觉氛围层面，体验柔和，反馈却缺少个体化。真正能帮助使用者进入状态的，是"自身生理节律被准确感知并被即时回应"——所以整个项目围绕一个工程问题展开：如何把看不见的呼吸与自主神经状态，转成可被身体理解的空间反馈。

体验闭环

Experience Loop

进入装置 → 被感知 → 获得反馈 → 节律趋稳 → 状态改善 → 数据验证。一个完整闭环，每个环节对应明确的技术实现。

System architecture

从身体信号
到空间响应。

装置拆成五层：感知、解算、通信、表现、评估。分层架构让系统可以在原型阶段快速替换输入方式和输出媒介——接触式与非接触式方案共存，体验反馈与验证机制同时成立。
Architecture

五层栈

  • 感知层 — 腰带压力传感 / HRV 手环 / 毫米波雷达

Developer thinking

关键判断，
公开展示。

展示结果，也展示决策。交互装置里真正有价值的，是为什么在某个阶段选择某种方案、为什么保留某条支线、为什么把美学判断和工程约束放在一起权衡。
A

接触式 / 非接触式共存

毫米波雷达减少穿戴摩擦，让体验更自然——适合最终装置。HRV 手环回答另一个问题：体验是否有效。一个降低交互门槛，一个提高证据密度。

B

把"疗愈感"翻译成工程指标

氛围、沉浸、放松——这些感性描述需要对应到可跟踪变量：呼吸频率是否趋于稳定、视觉响应延迟是否足够低、伪影是否会破坏体验连续性、HRV 是否出现副交感活跃度相关的积极变化。感性目标翻译成技术指标，项目才能被推进和迭代。

C

空间反馈需要节律逻辑

视觉系统对噪声、抖动和过度敏感做了抑制。反馈像呼吸一样有张有弛——视觉平滑度、灯光渐变曲线、数据置信度阈值，全部属于体验设计的工程表达。

美学选择

Why orange on black

黑色承载安静，橙色像热量、脉搏和被点亮的神经信号。同时具有仪器感和情绪温度。

工程选择

Why modular layers

原型阶段输入设备和输出媒介都在迭代。分层架构让系统替换成本更低，验证路径更清晰。

Design principle

美学建立吸引力。产品思维组织体验路径。工程思维让一切可运行、可测量、可迭代。

Validation

它有效吗？

多数交互装置到"体验很好"就结束了。这个项目多做了一步：HRV 手环纳入验证闭环，观察装置与焦虑缓解、副交感神经活跃度提升之间的关联。在最终版本中，HRV 继续作为系统的评估接口存在。
01

Experience Layer

毫米波雷达与空间视觉负责无负担的体验入口，降低穿戴摩擦，让使用者自然进入状态。

02

Validation Layer

HRV 手环提供时域生理指标，检测副交感活跃度是否在体验过程中出现积极变化。

03

Product Layer

体验与验证同时纳入，装置从"氛围设备"升级为"可评估的身心交互界面"。

效能验证

HRV 是证据层。

使用者在冥想垫上体验，旁边电脑显示实时生理曲线。手环用于检测装置对焦虑缓解和副交感神经活跃度的影响。

输出层

验证层回答"是否有效"，视觉层回答"如何被感知"。

Version evolution

从原型到系统。

项目经历了产品概念设计、硬件验证、呼吸灯体验测试、HRV 有效性探索，再到毫米波雷达与 TouchDesigner 视觉联动。每个版本回答更具体的问题。
V1 / Concept
空间冥想 → 产品原型
结构设计、场景表达、交互路径与物理形态。核心叙事："身体进入一个被回应的空间"。
V1 / Sensing
腰带压力映射 + HRV 手环验证
穿戴式输入探索呼吸与状态反馈的映射关系。同步引入 HRV 手环建立焦虑缓解和副交感活跃度的验证方向。
V2 / Radar
毫米波雷达 + ESP32 + 蓝牙数据流
非接触式呼吸检测。嵌入式侧完成滤波、呼吸率估计、置信度与伪影处理，JSON 数据送往外部视觉系统。
V2 / Spatial
TouchDesigner 视觉联动 + 空间沉浸
感知数据扩展为环境反馈——改变整个空间的呼吸节律，使用者与装置形成共振。
